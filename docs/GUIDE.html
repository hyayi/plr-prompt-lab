plr-prompt-lab — 사용 가이드

PLR(객체 속성 추출) 프롬프트를 측정하며 개선하는 독립 실험 도구 — PLR 전용. GPU·DB·Redis 없이 mock 경로가 전부 동작하고, 실측만 GPU가 필요합니다. 구조 시각화는 STRUCTURE.html 참고. (2026-07-02 · plr_v1.5_cot 강제커밋 · 검색 파이프라인 제거됨)

1 30초 시작 — GPU 없이 전체 루프 체험

python3 lab.py demo          # mock 모델로 run→eval→Δ 전체 사이클
python3 -m pytest tests/ -q  # 83 passed 확인

2 실험 사이클

데이터셋 준비 ──► lab run ──► lab eval ──► lab report / lab gallery
 (템플릿+라벨)    (GPU 1회/크롭)  (지표+ledger)   (비교표·confusion / 크롭 시각화)
                                                      │
              lab port ◄── prompts/<신버전>.yaml ◄── /improve-prompt 스킬
              (core/ir 반납)                          (근거 있는 개선안)

3 명령어

명령역할
lab demoGPU-free 온보딩 (mock 전체 사이클)
lab build-golden --video V --attribute A운영 비디오에서 골든셋 크롭 생성 (운영자 단계)
lab label --dataset D --female-in-male M3,M7 --unknown M9사람 라벨 (판별 불가는 unknown → 채점 제외·별도 집계)
lab validate-dataset --dataset D데이터셋 형식 검증 (fail-loud)
lab run --attribute A --version V --dataset D [--model gemma|mock]재채점 — 크롭당 모델 1회, margin·quality 기록
lab eval --attribute A --dataset Dacc·recall·precision·F1·confusion·bias·pred_unknown + margin/quality 캘리브레이션 + ledger Δ
lab experiment run exp.yaml교차곱 매트릭스 (datasets×models×prompts×attributes×formats×reasons)
lab report [--compare LEDGER_B]자체완결 HTML — 전체 실험 비교표·추이·히트맵·confusion (+실험군 간 비교)
lab gallery --dataset D크롭-라벨 시각화 HTML — 오답 우선, CORRECT/WRONG 배지, margin/quality
lab port [--apply]lab ↔ core/ir 프롬프트 표면 diff / 반납

4 프롬프트 — 두 소스, 한 가지 규칙

기본(--version 없음) = plr_prompts.py 상수 (현재 plr_v1.5_cot 강제커밋 — unknown 없음, 저신뢰는 margins로). 버전 지정 = lab run --version V가 prompts/V.yaml을 로드 — 실험은 항상 이 경로로. 새 버전은 yaml만 추가, 채택 시 PROMPT_VERSION_YAML_COT bump가 승격의 전부 → lab port.

5 데이터셋 — 구조는 고정, 라벨은 자유

my_dataset/
    manifest.yaml    # attribute·labels·pred_path 선언 (examples/dataset_template/ 복사)
    labels.jsonl     # {"obj_id", "label"} — 판별 불가는 "unknown"
    crops/<obj_id>.jpg
커스텀 속성은 manifest에 labels: [helmet, no_helmet] + pred_path: attributes.equipment[0].type처럼 선언하면 validate/run/eval이 그대로 동작. gender / vehicle_type / military는 내장 프리셋(선언 불필요)이자 참고 예시. 상세: docs/DATASET_SPEC.md

6 개선 루프 — /improve-prompt 스킬

분석자 → 제안자 → 비판자 ⇄ 수정자 → 판단자 (최대 3라운드). 분석자는 오답 크롭을 이미지로 직접 열람하고 패턴별 obj_id를 명명; 모든 제안은 본 샘플 + 수치 근거 필수; 판단자 합격 기준(측정 가능한 예상효과 등)을 통과해야 prompts/<신버전>.yaml 초안이 나옵니다. 상세: skills/improve-prompt/SKILL.md

7 실측(GPU) 전제조건

운영 ir 컨테이너가 GPU/VRAM을 점유 중 — 중지 후 실행하거나 오프피크에: docker-compose stop imageretrieval → lab run --model gemma … → start. 모델 준비는 docs/INSTALL.md.
