plr-prompt-lab — 실습 가이드 (인수인계)

PLR(객체 속성 추출) 프롬프트를 측정하며 개선하는 독립 실험 도구. 아래를 순서대로 그대로 복붙하면 모든 기능을 GPU 없이 체험합니다 — §7(실측)만 GPU가 필요합니다. 구조 시각화는 STRUCTURE.html. (2026-07-03 · plr_v1.5_cot 강제커밋 · 모든 명령 실행 검증됨)

1 demo — 전체 루프를 3초에 체험 (mock 모델)

python3 lab.py demo --keep    # --keep: 생성물(datasets/demo/)을 지우지 않고 남김
기대 결과: 합성 데이터셋 생성 → mock 재채점 2회 → 평가 → 두 버전의 Δ 출력. datasets/demo/에 crops/·labels.jsonl·predictions.jsonl·raw_responses.jsonl·ledger.jsonl·manifest.yaml이 남습니다.
MockModel이란: GPU 없이 미리 정해진 PLR YAML을 돌려주는 결정적 가짜 모델. 파이프라인 배관(재채점→파싱→평가→리포트)을 모델 없이 검증하는 용도이며, 프롬프트 품질과는 무관합니다 — 품질 측정은 §7의 실측으로.
자동 렌더링: §2·§3의 두 HTML은 lab eval·lab experiment가 끝날 때 자동 생성됩니다(데이터셋마다 gallery.html, ledger 옆 report.html — --no-render로 끔). 아래 수동 명령은 옵션을 바꿔 재생성할 때 씁니다.

5 내 데이터셋 만들기 — 다속성 라벨

cp -r examples/dataset_template datasets/my_test            # ① 템플릿 복사 (다속성 예제)
# ② crops/<obj_id>.jpg 채우고 manifest.yaml의 attributes: 맵 선언
# ③ labels.jsonl — 크롭당 한 줄, 속성별 정답을 dict로:
#    {"obj_id":"M3","labels":{"gender":"male","helmet":"helmet"}}
python3 lab.py validate-dataset --dataset datasets/my_test  # ④ 형식 검증 (fail-loud)
python3 lab.py run -X plr_v1.5_cot --dataset datasets/my_test              # ⑤ 모델 1회 — -A 불필요 (mock 가능)
python3 lab.py eval -A all --dataset datasets/my_test       # ⑥ 라벨 있는 전 속성 한 번에 채점 ★
모델 호출은 크롭당 1회 — run이 attributes.jsonl에 전체 plr_json을 저장하고, eval -A all이 속성별 예측을 pred_path로 재추출해 채점합니다 (GPU 재실행 없음). gender / vehicle_type / military는 내장 프리셋 (빈 dict로 선언), 커스텀 속성은 labels: + pred_path: 선언. 라벨 정책: 판별 불가 = unknown(채점 제외·별도 집계), 그 속성 키 자체를 빼면 미라벨(조인에서 제외). 단일 label 형식(구 데이터셋)도 계속 동작. gallery는 --attribute로 속성을 골라 배지를 그립니다. 사람·차량이 섞인 데이터셋은 행에 "object_type": "vehicle"을 적으면 그 크롭만 차량 프롬프트로 나갑니다. 상세: docs/DATASET_SPEC.md, 스킬: skills/prepare-dataset/SKILL.md

5⁺ 속성(-A) 레시피 — 경우의 수 전부

run — 모델은 속성 무관, -A는 predictions.jsonl 추출 뷰만 
python3 lab.py run -X plr_v1.5_cot --dataset datasets/my_test               # 표준: -A 생략 → manifest 첫 속성
[run] --attribute 생략 → manifest 첫 속성 'gender' 사용 (추출 뷰만 결정; 전체 속성은 attributes.jsonl에 저장됨)
python3 lab.py run -X plr_v1.5_cot --dataset datasets/my_test -A helmet     # 추출 뷰를 명시하고 싶을 때
python3 lab.py run -X plr_v1.5_cot -A gender                                # --dataset 생략 시엔 -A 필수 (eval/golden/<A> 규약 경로)
eval — 단일 / 쉼표 나열 / all 
python3 lab.py eval -A gender --dataset datasets/my_test          # 하나만 (라벨 고친 속성만 재채점할 때)
python3 lab.py eval -A gender,helmet --dataset datasets/my_test   # 골라서 — 나열 순서대로, 각각 ledger 1줄
python3 lab.py eval -A all --dataset datasets/my_test             # 라벨이 실제로 있는 속성 전부
all의 동작: 선언됐는데 라벨 없는 속성은 실패가 아니라 skip+안내(exit 0) — [eval] 라벨 없음 → skip: military (…). 라벨에는 있는데 manifest 미선언이면 WARNING(오타 방지). manifest 선언이 아예 없으면 라벨 키에서 직접 도출. 나열 중 하나가 실패해도 나머지는 계속 (마지막 요약 + exit 1).
gallery — 기본은 전부, 속성별 태그 + AND/OR 오답 필터 
python3 lab.py gallery --dataset datasets/my_test                 # 라벨된 속성 전부 — 카드마다 속성별 태그
python3 lab.py gallery --dataset datasets/my_test -A gender,helmet # 부분 집합만
python3 lab.py gallery --dataset datasets/my_test -A helmet       # 하나만 → 단일 배지 화면
다속성 화면의 필터바: [전체|오답만|정답만] × 속성 체크박스 × [OR(하나라도)|AND(모두)]. 예) "오답만 + gender·helmet 체크 + AND" = 두 속성 다 틀린 크롭만. 정렬은 오답 많은 카드 먼저, 저-margin 순. 실물 예시: gallery_multi_example.html
사람·차량 혼합 데이터셋 — 행 단위 object_type이 프롬프트를 라우팅 
# labels.jsonl — 크롭마다 종을 적으면 run 한 번에 각자 맞는 프롬프트가 나간다:
{"obj_id": "p1", "object_type": "person",  "labels": {"gender": "female"}}
{"obj_id": "v1", "object_type": "vehicle", "labels": {"vehicle_type": "sedan"}}

python3 lab.py run -X plr_v1.5_cot --dataset datasets/mixed   # p1→person 프롬프트, v1→vehicle 프롬프트
python3 lab.py eval -A all --dataset datasets/mixed           # gender는 사람 크롭에서, vehicle_type은 차량 크롭에서
부분 라벨 — 세 상태의 구분 
{"obj_id": "a", "labels": {"gender": "male", "helmet": "helmet"}}    # 둘 다 채점 참여
{"obj_id": "b", "labels": {"gender": "unknown", "helmet": "helmet"}} # gender: 사람도 판별불가 → 채점 제외+별도 집계
{"obj_id": "c", "labels": {"helmet": "helmet"}}                      # gender: 미라벨 → 조인에서 자연 제외 (아직 안 단 것)
